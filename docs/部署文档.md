# YPH 电商系统部署文档

## 环境要求

### 开发环境
- Python 3.8+
- MySQL 8.0+
- Redis 5.0+
- Node.js 16+ (前端)

### 系统要求
- Linux (推荐 Ubuntu 20.04+) 或 Windows
- 内存 2GB+
- 硬盘 20GB+

## 后端部署

### 1. 克隆项目
```bash
git clone <your-repo-url>
cd yph
```

### 2. 创建虚拟环境
```bash
cd backend
python -m venv venv

# Linux/Mac
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3. 安装依赖
```bash
pip install -r requirements.txt
```

### 4. 配置数据库
修改 `yph/settings.py` 中的数据库配置：

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'yph_ecommerce',
        'USER': 'your_db_user',
        'PASSWORD': 'your_db_password',
        'HOST': 'localhost',
        'PORT': '3306',
    }
}
```

### 5. 创建数据库
```sql
CREATE DATABASE yph_ecommerce CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 6. 执行数据库迁移
```bash
python manage.py makemigrations
python manage.py migrate
```

### 7. 创建超级用户
```bash
python manage.py createsuperuser
```

### 8. 收集静态文件
```bash
python manage.py collectstatic
```

### 9. 启动开发服务器
```bash
python manage.py runserver 0.0.0.0:8000
```

## 生产环境部署

### 使用 Gunicorn + Nginx

#### 1. 安装 Gunicorn
```bash
pip install gunicorn
```

#### 2. 创建 Gunicorn 配置
创建 `gunicorn_config.py`:

```python
bind = "127.0.0.1:8000"
workers = 4
worker_class = "sync"
timeout = 30
accesslog = "logs/gunicorn_access.log"
errorlog = "logs/gunicorn_error.log"
loglevel = "info"
```

#### 3. 启动 Gunicorn
```bash
gunicorn -c gunicorn_config.py yph.wsgi:application
```

#### 4. 配置 Nginx

创建 `/etc/nginx/sites-available/yph`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 20M;

    location /static/ {
        alias /path/to/yph/backend/static/;
    }

    location /media/ {
        alias /path/to/yph/backend/media/;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/yph /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 使用 Supervisor 管理进程

创建 `/etc/supervisor/conf.d/yph.conf`:

```ini
[program:yph]
command=/path/to/venv/bin/gunicorn -c gunicorn_config.py yph.wsgi:application
directory=/path/to/yph/backend
user=www-data
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/yph/gunicorn.log
```

启动：
```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start yph
```

## Redis 配置

安装 Redis:
```bash
# Ubuntu
sudo apt-get install redis-server

# 启动 Redis
sudo systemctl start redis
sudo systemctl enable redis
```

## 前端部署

### 开发环境
```bash
cd frontend/admin
npm install
npm run dev
```

### 生产构建
```bash
npm run build
```

构建产物在 `dist/` 目录，可以部署到 Nginx 或其他静态服务器。

## 常见问题

### 1. 数据库连接错误
检查MySQL服务是否启动，用户名密码是否正确

### 2. Redis连接错误
检查Redis服务是否启动

### 3. 静态文件404
运行 `python manage.py collectstatic`

### 4. 权限错误
确保 media 和 logs 目录有写入权限

## 性能优化建议

1. 启用 Redis 缓存
2. 使用 CDN 分发静态资源
3. 配置数据库索引
4. 启用 Gzip 压缩
5. 使用负载均衡

## 安全建议

1. 修改 SECRET_KEY
2. 设置 DEBUG = False
3. 配置 ALLOWED_HOSTS
4. 使用 HTTPS
5. 定期更新依赖包
6. 配置防火墙
